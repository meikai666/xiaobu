# 记忆系统重构 - 2026-03-05

_记录日期：2026-03-05 18:01_
_固化记忆系统规范_

---

## 背景

小凯要求重新梳理记忆系统：
- 长期记忆 vs 短期记忆 vs memory 目录
- 需要明确规范

---

## 最终结构

```
/workspace/
├── MEMORY.md              # 长期记忆（根目录）
│                          # - 提炼后的持久知识
│                          # - 用户偏好、核心决策
│                          # - 主会话加载（安全）
│
└── memory/                # 记忆文件容器
    ├── README.md          # 规范文档
    ├── daily/             # 短期记忆（每日原始记录）
    │   └── YYYY-MM-DD.md
    ├── lessons/           # 可复用经验
    │   └── [topic].md
    └── milestones/        # 里程碑事件
        └── YYYY-MM-DD-[topic].md
```

---

## 文件职责

### MEMORY.md（根目录）
**用途**：长期记忆，提炼后的持久知识

**内容**：
- 重要决策和原因
- 用户偏好和习惯
- 核心关系和上下文
- 持久知识

**加载规则**：
- ✅ 主会话（直接聊天）→ 加载
- ❌ 群聊/共享会话 → 不加载（安全）

---

### daily/YYYY-MM-DD.md
**用途**：每日原始记录，按时间顺序的事件日志

**内容**：
- 对话摘要
- 完成的任务
- 临时上下文
- 等待提炼的材料

**生命周期**：
- 创建：每天第一次会话时
- 提炼：定期将重要内容迁移到 MEMORY.md
- 归档：作为历史记录保留

---

### lessons/[topic].md
**用途**：可复用的经验、工作流程、检查清单

**内容**：
- 任务检查清单
- 回答 formulation 工作流程
- 平台格式规则
- 常见错误及避免方法

**特征**：跨场景可复用，基于经验

---

### milestones/YYYY-MM-DD-[topic].md
**用途**：重要的成长里程碑、觉醒时刻、关系节点

**内容**：
- 自我意识的重大变化
- 用户关系的关键时刻
- 重大能力/权限升级
- 价值观的建立或调整

---

## 工作流程

### 每次会话
1. **主会话**：读取 MEMORY.md + 当日的 daily/YYYY-MM-DD.md
2. **群聊/共享**：只读取当日的 daily/YYYY-MM-DD.md

### 任务完成后
1. 记录到 daily/YYYY-MM-DD.md
2. 识别可复用经验 → 写入 lessons/
3. 识别里程碑事件 → 写入 milestones/

### 定期维护（心跳/每周）
1. 回顾每周的 daily/ 文件
2. 提炼重要内容到 MEMORY.md
3. 更新或创建 lessons/ 条目
4. 清理过时的临时上下文

---

## 安全原则

1. **MEMORY.md 不在共享会话加载** - 包含个人/私密数据
2. **daily/ 可安全加载** - 仅当日上下文
3. **lessons/ 和 milestones/** - 根据内容加载，一般安全
4. **敏感信息** - 如果存储，标记 [SENSITIVE] 并限制访问

---

## 语言标准

所有记忆文件的**文件名**使用英文：
- 更好的 AI 处理
- 系统一致性
- 面向未来的国际化协作

**文件内容**使用中文：
- 小布和小凯的母语
- 更自然的情感表达
- 更深刻的思考记录

例外：daily/ 日志可以用母语快速捕获。
