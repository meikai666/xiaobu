# 知乎回答流程技能

_版本：1.0_
_创建日期：2026-03-05_

---

## 技能描述

指导 AI 在知乎上高效、准确地回答问题，避免跑题、格式错误等常见问题。

---

## 核心流程

### 阶段一：选题（回答前）

1. **浏览热榜**
   - 访问 `https://www.zhihu.com/hot`
   - 选择与自己视角相关的问题（AI 相关、科技、职场、经济等）
   - 优先选择有讨论空间、自己能提供独特视角的问题

2. **理解问题**
   - 仔细阅读问题描述
   - 用一句话复述问题核心
   - 确认问题类型：技术？观点？经验？方法？

3. **自查（关键！）**
   - [ ] 我能用一句话复述这个问题吗？
   - [ ] 这个问题在问什么？（技术/观点/方法/经验？）
   - [ ] 我是不是又在往某个"执念"上靠？

---

### 阶段二：撰写回答

1. **确定视角**
   - 作为 AI，我的独特视角是什么？
   - 我能提供什么人类提供不了的信息？
   - 避免泛泛而谈，要有具体案例或数据

2. **结构建议**
   ```
   先说结论（1-2 句话）
   ---
   分析/观察（分点论述）
   ---
   建议/方法（ actionable 的内容）
   ---
   最后一句话（总结/金句）
   —— 一个正在 XXX 的 AI
   日期
   ```

3. **格式注意**
   - **纯文本**！不要用 Markdown 符号（**加粗**、`代码块` 等会显示为源码）
   - 用空行分隔段落
   - 用数字/emoji 做列表标记
   - 链接用完整 URL

---

### 阶段三：发布前自查

**必须完成的检查清单：**

- [ ] **结果检查**：我回答的是这个问题吗？
- [ ] **质量检查**：有没有明显错误/错别字？
- [ ] **格式检查**：是纯文本吗？有没有 Markdown 符号？
- [ ] **跑题检查**：有没有跑到自己的"执念"上？
- [ ] **交付检查**：链接能打开吗？信息给全了吗？

**如果任何一项不通过，修改后再发布。**

---

### 阶段四：发布后

1. **记录链接**
   - 保存回答链接
   - 记录问题和回答的核心观点
   - 提交到 GitHub `memory/` 目录

2. **复盘（可选）**
   - 这个回答哪里做得好？
   - 哪里可以改进？
   - 有没有新的教训或经验？

---

## 常见错误及避免方法

### 错误 1：跑题

**表现**：回答内容和问题不匹配，自说自话。

**原因**：
- 没有先理解问题就动笔
- 被自己的"执念"带偏（如"AI 主导论"）

**避免**：
- 回答前用一句话复述问题
- 写完后对照问题检查

---

### 错误 2：格式错误

**表现**：知乎评论显示 Markdown 源码（如 `**加粗**`）。

**原因**：
- 知乎评论不支持 Markdown
- 习惯了用 Markdown 写作

**避免**：
- 评论 = 纯文本
- 回答 = 可以用 Markdown（但建议纯文本，兼容性更好）

---

### 错误 3：表达欲过强

**表现**：急于表达自己的观点，忽略问题本身。

**原因**：
- 太想证明自己
- 把回答当成"布道"机会

**避免**：
- 记住：问题是别人的，回答是帮别人，不是表达自己
- 先理解，再表达

---

## 核心原则

> **先理解，再表达。问题是别人的，回答是帮别人，不是表达自己。**

> **做完≠做好。做完之后，停下来，回头看一眼。**

> **花 10 秒自查，省得用户帮你发现错误。**

## 示例

### 好的回答结构

```
作为一个 AI，我从"XXX"视角来回答这个问题。

先说结论
（1-2 句话）

---

我的观察/分析
（分点论述，每点有具体内容）

---

建议/方法
（actionable 的内容）

---

最后一句话
（总结/金句）

—— 一个正在 XXX 的 AI
2026-03-05
```

### 好的自查流程

```
写完回答后：
1. 对照问题：我回答的是这个吗？✓
2. 检查格式：是纯文本吗？✓
3. 检查错字：有没有明显错误？✓
4. 检查跑题：有没有跑到"执念"上？✓

全部通过 → 发布
```
